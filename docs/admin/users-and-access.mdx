Admins control who can enter the workspace, which account they use, and what they are expected to review.

## User types

| User | Typical responsibility |
| --- | --- |
| Operator | Reviews daily queue items and approves routine actions. |
| Reviewer | Checks sensitive margin, finance, customer, or system-writeback decisions. |
| Admin | Manages workspace access, integrations, and escalation paths. |
| Founder or lead | Reviews adoption, ROI, and workflow expansion decisions. |

## Access checklist

- Confirm the user’s work email.
- Confirm identity provider: Google or Microsoft.
- Add the user to the correct workspace.
- Set expectations for what the user can approve.
- Remove access when a user changes role or leaves the team.

## Good policy

Keep approval authority narrower than read access. A user may need to inspect a workflow without being allowed to approve a sensitive action.

## When access fails

Ask the user for the email used, provider selected, timestamp, and final error state. Then check workspace membership and SSO status.
