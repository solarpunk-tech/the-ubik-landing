Ask Ubik when you need to recover context without chasing five tabs or asking another operator for a status update.

## Best uses

- Find what changed in an account, shipment, or quote.
- Recover decisions from meetings or long threads.
- Ask what is blocking a task.
- Pull together source context before approving a buyer reply.
- Help a manager understand why work is waiting.

## How to ask

<div className="ubik-steps">
  <div className="ubik-step"><span>01 · Name the object</span> Mention the buyer, shipment, account, PO, SKU, meeting, or thread.</div>
  <div className="ubik-step"><span>02 · Name the decision</span> Ask what you need to decide, not just what happened.</div>
  <div className="ubik-step"><span>03 · Ask for sources</span> Tell Ubik to show the email, document, meeting, or system context behind the answer.</div>
  <div className="ubik-step"><span>04 · Turn it into action</span> Ask for a draft reply, task, approval request, or owner handoff only after context is clear.</div>
</div>

## Useful prompts

| Situation | Ask Ubik |
| --- | --- |
| Buyer follow-up | "What has changed on this buyer request since the last quote?" |
| Shipment movement | "Which promise dates are affected by this ETA change?" |
| Manager review | "Which queue items have been blocked the longest and who owns them?" |
| Meeting memory | "What decisions came out of the supplier review call?" |
| Approval prep | "Show the source context I should check before approving this reply." |

## Reading an answer

A useful answer should include the conclusion, source context, uncertainty, and next step. If any of those are missing, ask a follow-up before acting.
